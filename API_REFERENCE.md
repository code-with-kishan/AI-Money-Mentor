# 📡 API Reference — AI Money Mentor Backend

Complete documentation of all backend API endpoints with request/response examples.

---

## Base URL

**Development**: `http://localhost:3000/api`
**Production**: `https://your-domain.com/api`

## Authentication

Currently, all endpoints are **public** (no authentication required). For production, add JWT middleware if needed.

## Error Handling

All errors return a 500 status with error message:

```json
{
  "error": "Failed to process request."
}
```

Errors are logged to console. For debugging, check server logs.

## 1️⃣ Financial Analysis

### POST `/api/analyze`

Calculates Money Health Score and generates AI insights.

**Request**:
```json
{
  "name": "Aarav Sharma",
  "age": 29,
  "salary": 1800000,
  "expenses": 70000,
  "investments": 850000,
  "monthlySavings": 87000
}
```

**Response** (200):
```json
{
  "score": 78,
  "savingsRate": 58.2,
  "emergencyMonths": 12.1,
  "insights": [
    "Savings rate is 58.2% based on salary vs monthly surplus.",
    "Expense to income ratio is 46.7%.",
    "Current investment corpus is INR 8,50,000."
  ],
  "recommendations": [
    "Set automated SIP to at least 25% of monthly take-home.",
    "Create an emergency reserve equal to 6 months of expenses.",
    "Review insurance coverage and nominee data this month."
  ]
}
```

**Field Descriptions**:
- `score`: 0-100 health score (higher = better)
- `savingsRate`: % of monthly income saved
- `emergencyMonths`: How many months expenses are covered by investments
- `insights`: AI-generated observations about spending/saving patterns
- `recommendations`: Actionable next steps for 90 days

**Error Cases**:
- Missing required fields → 400 Bad Request
- MongoDB down → Recommendations still returned (fallback mode)
- OpenAI timeout → Uses pre-computed fallback insights

## 2️⃣ AI Chat

### POST `/api/chat`

Chat with AI financial advisor.

**Request**:
```json
{
  "messages": [
    {
      "role": "user",
      "content": "How much should I invest monthly?"
    },
    {
      "role": "assistant",
      "content": "Based on your income..."
    }
  ]
}
```

**Response** (200):
```json
{
  "response": "Invest 25-30% of your monthly take-home after fixed expenses. With ₹18L salary and ₹70K expenses, you have ₹87K monthly surplus—invest ₹22-26K in diversified instruments (SIP, PPF, stocks)."
}
```

**Chat History**:
- Messages are saved to MongoDB `ChatHistory` collection
- If MongoDB is down, chat still works but history is lost on refresh

```json
{
  "response": "Demo AI mode: Focus on reducing fixed monthly costs, automate SIPs, and keep emergency savings for 6 months."
}
```

**Notes**:
- Each message is processed independently
- No conversation memory between requests (pass full history client-side)
- Temperature set to 0.3 for consistent, factual responses

## 3️⃣ SIP Calculator

### POST `/api/sip`

Calculate SIP maturity using compound interest formula.

**Request**:
```json
{
  "monthlyInvestment": 15000,
  "annualReturnRate": 12,
  "years": 10
}
```

**Response** (200):
```json
{
  "investedAmount": 1800000,
  "estimatedReturns": 1983742,
  "maturityAmount": 3783742
}
```

**Formula Used**:
```
M = P × [((1+i)^n - 1) / i] × (1+i)
- P: Monthly investment
- i: Monthly return (annualReturnRate / 12 / 100)
- n: Total months
```

**Validation**:
- `monthlyInvestment` > 0
- `annualReturnRate` ≥ 0
- `years` > 0

**Edge Cases**:
- 0% return: Maturity = invested amount + fees
- Negative values: Treated as 0, no error thrown
- Very large numbers: Capped at JavaScript number limits

## 4️⃣ Tax Optimizer

### POST `/api/tax`

Compare Indian old vs new tax regimes (2024-25).

**Request**:
```json
{
  "annualSalary": 1800000,
  "deductions": 220000
}
```

**Response** (200):
```json
{
  "oldRegimeTax": 220000,
  "newRegimeTax": 237600,
  "betterRegime": "old",
  "taxSaving": 17600,
  "summary": "The OLD regime appears better with estimated savings of INR 17,600.",
  "actions": [
    "Review 80C and NPS contributions before final filing.",
    "Use old regime only if deductions are consistently high.",
    "Re-evaluate tax choice every financial year."
  ]
}
```

**Tax Slabs** (FY 2024-25):

**Old Regime** (with ₹50K standard deduction):
```
₹0 - ₹2.5L: 0%
₹2.5L - ₹5L: 5%
₹5L - ₹10L: 20%
₹10L+: 30%
+ 4% cess on total tax
```

**New Regime** (with ₹75K standard deduction):
```
₹0 - ₹3L: 0%
₹3L - ₹6L: 5%
₹6L - ₹9L: 10%
₹9L - ₹12L: 15%
₹12L - ₹15L: 20%
₹15L+: 30%
+ 4% cess on total tax
```

**AI Advisory**:
- Generated by LangChain based on comparison
- Includes recommendations for deductions
- Fallback: Static advice about annual re-evaluation

## 5️⃣ Wealth Simulator

### POST `/api/simulate`

Project wealth growth with monthly contributions.

**Request**:
```json
{
  "currentCorpus": 600000,
  "monthlyContribution": 25000,
  "annualReturnRate": 11,
  "years": 15
}
```

**Response** (200):
```json
{
  "finalCorpus": 8650000,
  "projections": [
    { "year": 1, "value": 925000 },
    { "year": 2, "value": 1275000 },
    ...
    { "year": 15, "value": 8650000 }
  ],
  "narrative": "Projected corpus may reach INR 86,50,000 in 15 years at 11% expected annual return.",
  "milestones": [
    "Increase monthly contribution by 10% annually to accelerate corpus growth.",
    "Review equity-debt allocation every 12 months.",
    "Use goal-based buckets for education, house, and retirement."
  ]
}
```

**Projection Algorithm**:
```
Each year:
A = P(1+r)^n + C × [((1+r)^n - 1) / r] × (1+r)
- P: Previous year corpus
- C: Monthly contribution
- r: Monthly return rate
- n: 12 months
```

**Output Frequency**:
- Data points exported **yearly** (simpler client charts)
- Internal calculation is monthly

**Use Cases**:
- Retirement planning (25-40 year projection)
- Goal funding (5-10 year projection)
- Home down payment (3-5 year projection)

## 6️⃣ Portfolio Analyzer

### POST `/api/portfolio`

Analyze holdings and generate risk assessment.

**Request**:
```json
{
  "holdings": [
    { "asset": "Nifty 50 ETF", "amount": 220000, "category": "Equity" },
    { "asset": "PPF", "amount": 120000, "category": "Debt" },
    { "asset": "Gold ETF", "amount": 80000, "category": "Gold" }
  ]
}
```

**Response** (200):
```json
{
  "totalValue": 420000,
  "categorySplit": {
    "Equity": 220000,
    "Debt": 120000,
    "Gold": 80000
  },
  "riskLevel": "moderate",
  "notes": [
    "Diversify across equity, debt, and gold for downside control.",
    "Keep single-asset concentration below 30% where possible.",
    "Match risky allocation with goal timeline and risk capacity."
  ]
}
```

**Risk Classification**:
```
Equity % ≤ 45%  → low risk
Equity % 45-75% → moderate risk
Equity % > 75%  → high risk
```

**Category Support**:
- `Equity`: Stocks, mutual funds, ETFs
- `Debt`: Bonds, PPF, RDs, fixed deposits
- `Gold`: Gold ETFs, physical gold
- `Cash`: Money market, savings account
- `Others`: Crypto, real estate (manual adjust needed)

**AI Notes**:
- Generated by portfolio agent
- 2-6 specific recommendations
- Fallback: Generic diversification advice

## 🔄 Common Workflows

### 1. First-Time User Flow
```
1. GET /api/demo → Load sample data
2. POST /api/analyze → Calculate health score
3. Dashboard displays results
4. User can modify data and recalculate
```

### 2. Investment Planning Flow
```
1. POST /api/sip → Get maturity amount
2. POST /api/simulate → Project wealth over time
3. Combine with POST /api/tax → See tax-efficient allocation
```

### 3. Tax Filing Prep Flow
```
1. POST /api/tax → Compare regimes
2. POST /api/analyze → See overall financial health
3. Recommend regime choice + deductions to claim
```

### 4. Conversational Planning
```
1. POST /api/chat → Ask question
2. POST /api/analyze → Get score
3. POST /api/chat → Follow-up question
4. Repeat as needed
```

## 🚨 Rate Limiting

**Currently**: No rate limiting enabled.

**For Production**, add middleware:
```javascript
// Example: 100 requests per 15 minutes per IP
import rateLimit from 'express-rate-limit';

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 100,
});

app.use('/api/', limiter);
```

## 🔐 Input Validation

All endpoints validate inputs server-side. Examples:

```javascript
if (!body.salary || body.salary <= 0) {
  return NextResponse.json({error: "Invalid salary"}, {status: 400});
}

if (!body.annualReturnRate || body.annualReturnRate < 0 || body.annualReturnRate > 100) {
  return NextResponse.json({error: "Return rate must be 0-100%"}, {status: 400});
}
```
